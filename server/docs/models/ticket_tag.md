## TicketTag

The model's columns and their descriptions:

- id: Primary key
- ticket_id: Foreign key to the ticket
- tag_id: Foreign key to the tag

The model's relationships:

- ticket: Ticket associated with the tag
- tag: Tag associated with the ticket
